# Dashboard

The Dashboard is the "what deserves attention right now?" screen.

Open it at `/`.

## What it is for

Use the Dashboard when you want a fast daily review instead of digging through every page.

It combines:

- relationship insights
- project insights
- recent activity
- high-level totals

## What you see

### 1. Summary line

At the top, the page tells you how many open insights exist across relationships and projects.

This is your short answer to:

"Is there anything I should handle today?"

### 2. Stats row

The cards summarize your current system, such as:

- total contacts
- active projects
- open relationship insights
- open project insights

### 3. Action matrix

The center of the page uses an Eisenhower-style matrix:

```text
 +----------------------+----------------------+
 | Do First             | Schedule             |
 | Urgent + Important   | Not Urgent + Important |
 +----------------------+----------------------+
 | Delegate             | Eliminate            |
 | Urgent + Less Important | Low urgency/importance |
 +----------------------+----------------------+
```

secondbrain places insights into these buckets based on their priority and type.

This helps you avoid two common problems:

- reacting emotionally to the newest item
- missing something important because it was not loud

### 4. Recent activity

This section shows the latest project-related communications across your data sources.

Use it to answer:

- what moved recently?
- which project is waking up?
- what source is generating the most movement?

## Typical daily workflow

```text
 Open Dashboard
      |
      v
 Read open insight count
      |
      v
 Scan "Do First"
      |
      v
 Open the related project or person
      |
      v
 Take action or mark the insight handled
```

## Actions you can take

Depending on the item:

- mark a relationship insight as actioned
- dismiss a relationship insight
- resolve a project insight
- click through to the underlying relationship or project

## What "actioned", "dismissed", and "resolved" mean

These are not the same:

- `Actioned`: you did something about it
- `Dismissed`: you decided it is not useful
- `Resolved`: the project issue is no longer open

Use them honestly. The Dashboard gets cleaner and more trustworthy when you close items properly.

## When the Dashboard is most useful

The Dashboard shines when:

- you have more than one data source connected
- the agents have run at least once
- you are checking it regularly, not only during emergencies

## When the Dashboard may look empty

An empty Dashboard usually means one of these:

- the core agents have not run yet
- the source agents are not ingesting data
- you have already closed the current insights

An empty state is not always a problem. Sometimes it genuinely means you are caught up.

## Best habits

- check the Dashboard first, not your inbox first
- use it to decide where to spend attention
- click into details instead of trying to act from memory
- close insights once you have dealt with them

## What the Dashboard is not

The Dashboard is not a full record browser. It is a triage surface.

Think of it like this:

```text
 Dashboard = "What matters now?"
 Relationships = "Who matters?"
 Projects = "What is moving?"
 Groups = "What are my WhatsApp circles doing?"
 Search = "Where did this topic appear?"
```
