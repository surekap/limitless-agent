# Privacy, Trust, and Limits

This guide explains how to think responsibly about secondbrain.

## What kind of system this is

secondbrain is a private intelligence layer over highly sensitive information:

- email
- messages
- lifelogs
- stakeholder context
- AI-generated summaries

That means you should use it thoughtfully, especially if you run a business or manage confidential relationships.

## What to trust

Trust most:

- original communications
- manually corrected records
- recent activity
- clearly evidenced project state

Trust carefully:

- AI summaries
- inferred roles
- inferred opportunities
- soft relationship interpretations

Rule:

```text
 Evidence first
 AI interpretation second
 Judgment always last
```

## Sensitive use cases that need caution

Be careful with:

- legal matters
- HR matters
- compensation discussions
- sensitive investor conversations
- medical or deeply personal information
- client or customer information with strict confidentiality expectations

The system can store and surface useful context, but that does not remove your duty of care.

## Good safety habits

- correct important records manually
- validate before acting on sensitive insights
- avoid treating one summary as the whole truth
- use Search and communications to verify context
- review who has access to the machine and database

## What the system is not

secondbrain is not:

- a perfect memory
- a substitute for judgment
- a legally authoritative system of record
- a guaranteed truth engine

It is an amplifier. It amplifies both good and bad habits.

## What it is best at

It is strongest at:

- surfacing things you might miss
- giving fast context before a conversation
- keeping people and projects visible over time
- making fragmented signals easier to interpret

## What it is weaker at

It is weaker at:

- subtle emotional nuance
- political context that never appears in messages
- understanding sarcasm, omission, or private tensions
- making high-stakes judgments without human review

## Questions to ask before acting on an AI conclusion

- What evidence is this based on?
- Is there recent communication that supports it?
- Could this be a misclassification?
- Is this a real decision or only a prompt to investigate?

## Data minimization mindset

Just because the system can ingest something does not mean it should.

If you are unsure, ask:

- Does this belong here?
- Would I be comfortable if this were surfaced unexpectedly later?
- Does the business value outweigh the sensitivity?

## How to become superhuman safely

The safe version of "superhuman" is:

- better memory
- better preparation
- better timing
- better prioritization

It is not:

- blind faith in automation
- acting on unverified inferences
- outsourcing judgment

## The right relationship with the system

Use secondbrain like this:

```text
 It notices
 I verify
 I decide
 I act
```

That keeps the system useful, trustworthy, and sane over time.
